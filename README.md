# Graph Project V-2.0.0.

MVP интерфейс для визуализации, исследования и поэтапного расширения графовых данных в браузере.

Проект представляет собой рабочий стенд для пользовательских и технических сценариев анализа графа:
- размещение опорного узла
- выбор реального узла через поиск
- работа с mock-данными через MSW
- подключение к реальному backend API
- визуализация узлов и связей
- интерактивное расширение графа
- предварительный просмотр expand-операций
- фильтрация расширения по типам, направлениям и атрибутам
- просмотр информации по узлам и ребрам
- работа с 1-hop окружением
- создание и анализ кластеров
- фокусировка на текущей области
- ручной перезапуск layout

Backend-репозиторий: [PecheslavVenik/T-graph_2](https://github.com/PecheslavVenik/T-graph_2)

## Текущее состояние

Сейчас проект реализован как MVP для исследования графа перед дальнейшей интеграцией и развитием продукта. Основной акцент сделан на:
- отрисовку графа через Sigma.js
- управление сценой и камерой
- режим расширения графа
- режим кластерного анализа
- работу с mock API и реальным API
- интерактивные контекстные действия
- проверку layout-логики на разных объемах данных
- подготовку пользовательских сценариев для работы с backend

При запуске приложение предлагает выбрать источник данных:
- `Mock` - работа с локальными mock-данными через MSW
- `Реальный API` - работа с внешним backend API

## Возможности интерфейса

### Основной режим графа

В основном режиме доступна работа с текущим графом:
- размещение опорного узла
- поиск и выбор стартовой ноды
- отображение найденных узлов и связей
- фокусировка камеры на текущей области
- ручной перезапуск layout для всех нод
- управление hover-карточками для узлов и ребер
- настройка поведения фокуса при выборе элементов
- настройка динамического размера нод при zoom

### Контекстные действия

Для узлов доступны контекстные действия:
- просмотр информации
- просмотр 1-hop окружения
- предварительный просмотр расширения
- удаление узла из текущего представления

Для ребер доступен просмотр детальной информации по связи и агрегированным raw edges.

### Expand preview

Перед расширением графа можно открыть preview и настроить параметры expand-операции:
- направление обхода: `BOTH`, `OUTBOUND`, `INBOUND`
- фильтры по relation family
- фильтры по edge type
- фильтры по node type
- предустановленные фильтры по атрибутам
- лимит нод
- лимит связей
- настройки layout после расширения
- фокусировка камеры на результате расширения

### Drawer-панель

Боковая рабочая панель используется для:
- информации по узлу
- информации по ребру
- предварительного просмотра expand-операции
- отображения атрибутов, идентификаторов, статусов и технических данных

### Режим кластеров

В режиме кластеров доступны:
- создание кластера из 1-hop окружения
- список созданных кластеров
- выбор активного кластера
- фокусировка камеры на выбранном кластере
- hover-информация по кластеру
- удаление кластеров
- перезапуск layout только для выбранных кластеров

## Запуск проекта локально

Для локального запуска нужен установленный Node.js и npm.

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск dev-сервера

```bash
npm run dev
```

### 3. Адрес приложения

По умолчанию приложение будет доступно по адресу:

```text
http://localhost:5173
```

или

```text
http://127.0.0.1:5173
```

## Сборка проекта

Сборка production-версии:

```bash
npm run build
```

Локальный просмотр production-сборки:

```bash
npm run preview
```

## Запуск через Docker Compose

В проекте предусмотрен запуск через Docker Compose. Для этого варианта локальный Node.js не нужен.

Команда запуска:

```bash
docker compose up
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:5173
```

или

```text
http://127.0.0.1:5173
```
